## 데이터 흐름 (학생 한 명의 일생)

```
1️⃣ 최초 상담/등록
   학부모가 학원에 전화 → 엑셀에 기록 (16,000건 중 하나)
                              ↓
                     📦 contacts에 기본정보 저장
                        (이름, 연락처, 보호자명, 학교, 등록일)

2️⃣ 반 배정 (등원)
   학생이 HA101반에 배정됨
        ↓
   📦 students에 수업 정보 생성    +    📦 contacts에 [등원] 이력 추가
      (enrollments: HA101, 월수금)         ([등원] 2020-03-01 HA101)

3️⃣ 학기 중 수업
   📦 daily_checks에 매일 출결/숙제/테스트 기록 (DSC가 관리)

4️⃣ 퇴원
   학생이 그만둠
        ↓
   📦 students: 상태→퇴원        +    📦 contacts에 [퇴원] 이력 추가
                                        ([퇴원] 2020-12-15 HA103 사유:이사)

5️⃣ 재등원 (1년 후 다시 옴)
   이름/연락처 입력 → contacts에서 자동 검색 → 기본정보 자동 채움!
        ↓
   📦 students에 새 수업 추가    +    📦 contacts에 [재등원] 이력 추가

6️⃣ 매년 3월: 학년 승격
   초5 → 초6, 중3 → 고1 ... (한 번에 전체 승격)
```

## 핵심 용어 사전

| 용어 | 쉬운 설명 |
|------|----------|
| **Firestore** | 구글 클라우드에 있는 데이터 창고 |
| **컬렉션 (Collection)** | 창고 안의 서랍장 (students, contacts 등) |
| **문서 (Document)** | 서랍장 안의 파일 한 장 (학생 한 명의 정보) |
| **필드 (Field)** | 파일 안의 각 칸 (이름, 연락처, 학교 등) |
| **docId** | 파일에 붙이는 이름표 (예: "김민준_1012345678") |
| **enrollments** | 수업 목록 (한 학생이 듣는 반 정보들의 묶음) |
| **setDoc(merge)** | 기존 파일에 새 내용을 덮어쓰기 (기존 것은 유지) |
| **보안 규칙** | "이 서랍장은 선생님만 열 수 있어요" 같은 규칙 |
| **배포 (Deploy)** | 수정한 코드를 인터넷에 올려서 실제로 적용하는 것 |

## 각 서랍장(컬렉션)의 역할

### 📦 students (학생 관리)
- **누가 쓰나:** impact7DB 앱
- **뭐가 들어있나:** 현재 재원생의 수업 정보, 상태, 메모
- **언제 변하나:** 학기 초 CSV 업로드, 수동 등록/수정, 퇴원 처리

### 📦 contacts (전체 학생 명부) ⭐ 새로 만들 예정
- **누가 쓰나:** impact7DB + impact7DSC (둘 다)
- **뭐가 들어있나:** 15년간 모든 학생의 기본 인적정보 + 등퇴원 이력
- **언제 변하나:** 학생 등록/퇴원/재등원 시 자동 업데이트

### 📦 daily_checks (일일 체크)
- **누가 쓰나:** impact7DSC 앱
- **뭐가 들어있나:** 매일 출결, 숙제, 테스트 결과
- **언제 변하나:** 매일 선생님이 기록

### 📦 daily_records (일일 기록)
- **누가 쓰나:** impact7DSC 앱
- **뭐가 들어있나:** 수업 진행 기록
- **언제 변하나:** 매일 선생님이 기록

### 📦 history_logs (변경 이력)
- **누가 쓰나:** 자동 기록 (등록/수정/퇴원 시마다)
- **뭐가 들어있나:** 누가 언제 무엇을 변경했는지
- **언제 변하나:** 학생 데이터가 변경될 때마다 자동

## contacts 문서 상세 구조

하나의 학생 문서(파일) 안에 들어있는 내용:

```
📄 김민준_1012345678 (문서 이름표 = docId)
│
├── 기본 정보
│   ├── name: "김민준"              (학생 이름)
│   ├── student_phone: "1066476114" (학생 연락처)
│   ├── parent_phone_1: "1012345678" (부모 연락처1)
│   ├── parent_phone_2: "1098765432" (부모 연락처2)
│   ├── guardian_name_1: "김철수"    (보호자명1)
│   ├── guardian_name_2: "이영희"    (보호자명2)
│   ├── school: "대일고"             (학교)
│   ├── grade: "고1"                 (학년)
│   └── level: "고등"               (학부)
│
├── 날짜 정보
│   ├── first_registered: "2020-02-15"  (최초등록일 = 상담일)
│   └── first_attended: "2020-03-01"    (첫등원일 = 첫 반배정일)
│
└── 등퇴원 이력 (status_history)
    ├── [0] 등원   2020-03-01  HA101
    ├── [1] 퇴원   2020-12-15  HA103  사유: 타학원 이동
    ├── [2] 재등원 2022-03-01  I201
    ├── [3] 퇴원   2023-06-30  AX202  사유: 유학
    └── [4] 재등원 2025-01-05  HS101
```

## CSV/구글시트 헤더 매핑표

| 영어 (표준 헤더) | 한글 (호환 유지) | Firestore 필드 |
|---|---|---|
| name | 이름 | name |
| level | 학부 | level |
| school | 학교 | school |
| grade | 학년 | grade |
| student_phone | 학생연락처 | student_phone |
| parent_phone_1 | 학부모연락처1 | parent_phone_1 |
| parent_phone_2 | 학부모연락처2 | parent_phone_2 |
| guardian_name_1 | 보호자명1 | guardian_name_1 |
| guardian_name_2 | 보호자명2 | guardian_name_2 |
| branch | 소속 | branch |
| level_symbol | 레벨기호 | enrollments[].level_symbol |
| class_number | 반넘버 | enrollments[].class_number |
| class_type | 수업종류 | enrollments[].class_type |
| start_date | 시작일 | enrollments[].start_date |
| end_date | 종료일 | enrollments[].end_date |
| day | 요일 | enrollments[].day |
| status | 상태 | status |
| pause_start_date | 휴원시작일 | pause_start_date |
| pause_end_date | 휴원종료일 | pause_end_date |
| semester | 학기 | enrollments[].semester |

> 영어 헤더가 표준이며, 한글 헤더도 하위 호환을 위해 계속 지원됩니다.
